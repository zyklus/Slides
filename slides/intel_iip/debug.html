Other Debuggers
Both the Chrome and Internet Explorer debuggers have similar functionality to Firebug.

Google Chrome

Comes with a excellent debugger built in. To access it, go to View --> Developer --> Developer tools or press:
  • Windows: ctrl + shift + i
  • Mac: ⌥ + ⌘ + i

Internet Explorer

IE8 and IE9 both come with build in debuggers. To access, go to View --> Developer Tools or press:
  • F12
Note: Internet Explorer developer tools are buggy. It is strongly suggested that most debugging be done in either Firefox or Chrome, and the Internet Explorer debugging tools used to solve IE-specific issues.

One of the main issues is that IE does not display updated HTML since the debugger was opened. This means popup content, etc will not be displayed unless they were opened before the debugger was opened.
Reference: Download Google Chrome
Internet Explorer Rendering Modes
The IE9 developer tools (and possibly IE8 as well) allow you to easily switch between IE7/8/9 rendering modes. This means that you are viewing the page in IE9 as a previous browser (IE7 or IE8) would render it.

  • Do not change the document mode, only the browser mode
  • There is no need to ever test in "Quirks Mode" -- This was obsolete even in IE6
  • Although very good for quick testing of older browsers, switching rendering modes in IE9 is not a 100% accurate representation of how older browsers will render a page.
  • If absolutely necessary to test in actual instances of older versions of IE, you can use IETester (which can only be installed if you have not installed IE9)
  • IETester can be installed in a VM environment, likely in Windows XP, for the rare cases when it is necessary to use
Reference: Download IETester
Console Debugging

Echo basic messages

console.log( any_object );

Show a stack trace

console.trace()

Count calls to a specific event

console.count( event_title );

Group console statements

console.group();
console.groupEnd();
Note: Do not leave console statements in production code as it will cause JavaScript errors in browsers without debuggers enabled
Reference: Firebug Console API
