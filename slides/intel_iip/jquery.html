jQuery The $ Function
jQuery's $ function is overloaded in several ways:

As a selector

var elements = $('#myID span');

As a DOM Creation Utility

var node = $('<div id="myID"><span>Hello, World!<span></div>');

As an onload handler

$(function(){
  // Any code put here will fire when the document is loaded
});

jQuery Sets
Whenever selecting elements with the jQuery selector, e.g. $('#myID span'), jQuery returns a custom "array-like" object. This object is used to manipulate every element in this set at the same time. For example, if you wanted to hide every element on the page with a certain classname: 
$('.className').hide()
In certain instances, you need or want references to the actual DOM objects that the set is referencing: 
var nodes = $('.className');
nodes.get();  // returns an array of DOM Nodes
nodes.get(5); // returns the 6th element
nodes[5];     // returns the 6th element

Chaining

In most cases, jQuery methods return another jQuery object, so you can chain methods together: 
$('.className').show().addClass('active').animate({ left : 100 });

Getting Data

Whenever a jQuery function returns non-DOM Node data, e.g. .width() or .attr('id'), you are unable to chain because the method returns a primary JS object: 
$('.className').width(); // width, e.g. "100"

jQuery Event Listners
In JavaScript, most code is run in response to an event: The user clicks on a button, the user moves their mouse over a drop-down, etc.

Binding (Attaching) Event Handlers

$('.className').bind('click', function(ev){ /* .. */ });
$('.className').bind('change keypress keyup', function(ev){ /* .. */ });
$('.className').click(function(ev){ /* ... */ })

Un-Binding Event Handlers

$('.className').unbind('click', handlerFn /* Optional */);
// Note: handlerFn must reference the same function that was originally
         bound

"this"

$('.className').bind('click', function(ev){
	alert(this.id); // this === the DOM Node clicked on
});

Event object

The above references to ev in the function declarations refer to the jQuery event object, which normalizes event data like mouse position, and provides a way to stop event propagation and prevent default actions.

jQuery "Live" and "Delegate" Event Listners

Background

In many circumstances, it may be desired to have an event listener on many, many similar objects such as cells in a table or items in a list. In this case it is inefficient to attach an event listener to every element. Instead, an event listener can be attached to the parent element. In JavaScript, events propagate up the DOM tree until they are cancelled, so attaching a "click" listener, for example, to a table will catch all click events on every cell in that table. jQuery provides easy ways to do this:

Live

$('#tableID td').live('click', function(ev){
	// "this" refers to the td cell that was clicked on
});

Delegate

"Live" can sometimes be inefficient since the events are internally attached to the body and jQuery will be forced to check any click, anywhere on the page, to see if it was a match. Delegate solves this problem by attaching the event listener to the object specified: 
$('#tableID').delegate('td', 'click', function(ev){ ... });
Note: Unlike binding to a single object using .bind, .click, etc, .live and .delegate will function on elements that are added to the page after the event handler was attached. This is very useful for pages that are dynamically modified since event handlers do not need to be re-attached after every page modification.

jQuery Traversing the DOM
var nodes = $('.className');

nodes.closest ('selector'); // each element is replaced with the nearest
	                       div UP the tree, including itself
nodes.parent  ('selector'); // similar to closest; does not include self
nodes.find    ('selector'); // child elements that match selector
nodes.children();           // direct child elements
nodes.children('selector'); // direct child elements that match selector
nodes.filter  ('selector'); // current elements that match selector
nodes.filter  (function);   // filter current elements by function

jQuery Data
Partially due to various browsers (IE) having internal memory leak issues during garbage collection, jQuery provides an internal structure for storing data "on" DOM elements. Use this whenever it is necessary to have JavaScript data attached to a specific event. This becomes especially useful in conjunction with .live or .delegate:

Storing Data

var node = $('#someID');
node.data('key', 'value');

Retrieving Data

var node = $('#someID');
node.data('key');

node.data(); // returns an object containing all stored data

jQuery AJAX

AJAX in General

AJAX is a term coined that simply involves sending / receiving data between the client and server without reloading the entire page. From a client point-of-view, JavaScript asks for a page, receives a response, and processes it.

AJAX in jQuery

$.get( '/path/to/url', { key : value }, function(data){
  // handle data
});

$.post( ... );
$.ajax({ url: '/path/to/url', success: function(data){ ... } });

Asynchronous handling

Note that all jQuery AJAX functions are, by default, asynchronous. This means that JavaScript code will continue to run in the background while the HTTP request is processing. When the request is complete, a callback is triggered. If requests are done in a synchronous manner, all actions on the page get blocked (the user is unable to type, click, etc) until the request is finished. This is strongly discouraged. 
var x = 10;
$.get('/some/url', function(){
  x = 20;
});
// x == 10

jQuery Helper Functions
jQuery provides many helper functions that are meant to operate on Arrays and jQuery sets.

noop

Does nothing, is used as a "safe default": 
function(callback){
  (callback || $.noop)();
}

Each

$('div').each(function(){
  var $this = $(this);
  $this.addClass($this.width() > 200 ? 'large' : 'small');
});

Map

$('div').map(function(){
  return $(this).width();
});
// e.g. [100, 225, 42, 600]
